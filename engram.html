What is Engram

A memory layer, not a vector store

Most AI pipelines give each conversation a blank slate. Engram adds a structured, queryable, time-aware memory that persists across sessions. It stores facts as subject–predicate–object triples, not raw text blobs, so retrieval is precise and relationships are first-class.

🧠

Structured facts

Memories are stored as Alice → prefers → dark mode triples, not text. Query by entity, predicate, or full-text.

🕸️

Knowledge graph

Entities are linked. A 2-hop BFS traversal surfaces related facts that a keyword search would miss.

⏱️

Time-aware

Every fact has valid_from / valid_to. Updating a fact supersedes the old one — full history is preserved.

🔒

Scoped memory

Separate memory namespaces per user, session, or agent. Zero cross-contamination by default.

⚖️

Confidence & decay

Low-confidence facts are quarantined. Unused facts decay in strength; frequently accessed ones are promoted.

📌

Memory blocks

Always-surfaced key/value slots for agent-editable state: persona, current goal, user profile — pinned to every context call.

Hybrid scoring weights (defaults)

Each retrieved fact gets a composite score:

Semantic (embedding)0.40
Keyword (FTS5)0.30
Graph (BFS hop distance)0.10
Temporal (recency)0.10
Salience0.05
Strength (reinforcement)0.05

Graph score: g = 1 / (1 + hop × 0.5) — seed facts score 1.0, 1-hop 0.667, 2-hop 0.5.

Memory lifecycle

store() — ingestion

Text is passed through your extractor. Entities are created or resolved. Facts are inserted with truth_state = 'current'. Low-confidence facts go to quarantine.

retrieve() / getContext() — retrieval

FTS + graph BFS + temporal scoring returns the top-K most relevant facts, related entities, and source episodes. Retrieval traces are written and accessed facts are reinforced.

updateFact() — supersession

The old fact gets valid_to = now and truth_state = 'superseded'. A new fact is inserted. History is preserved — you can query asOf any date.

workers.runDecay() — strength decay

Infrequently accessed facts lose strength over time (power-law decay). High-heat facts (access × strength / log(age)) are flagged for promotion.

forget() — soft or hard delete

Soft forget hides a fact (still queryable via asOf). Hard forget removes it from the database permanently.

Syntax reference

API at a glance

Both Python and Node.js expose an identical high-level API. Scroll the tabs to explore every method.

// ── Node.js ──────────────────────────────────────────
import { Engram } from 'engram-node';

// Minimal — uses in-memory SQLite
const mem = new Engram({ dbPath: ':memory:' });

// Full config with a persistent file
const mem = new Engram({
  dbPath:                  'agent.db',
  defaultScope:            'user-42',

  // Confidence thresholds
  extractorConfidenceMin:  0.7,
  supersessionConfidenceMin: 0.85,

  // Hybrid scoring weights (must sum to 1.0)
  weightSemantic:          0.40,
  weightKeyword:           0.30,
  weightGraph:             0.10,
  weightTemporal:          0.10,
  weightSalience:          0.05,
  weightStrength:          0.05,

  graphMaxHops:            2,
  defaultTopK:             10,
});

// Resource cleanup (Node.js ≥23 'using' keyword)
{
  using mem = new Engram({ dbPath: ':memory:' });
  // mem.close() called automatically at end of block
}

// ── Python ───────────────────────────────────────────
from engram import Engram

mem = Engram(db_path=":memory:")

mem = Engram(
    db_path="agent.db",
    default_scope="user-42",
    extractor_confidence_min=0.7,
    weight_semantic=0.40,
    graph_max_hops=2,
)
// store() — persist facts from text or structured data
// Requires a custom extractor to extract entities/facts from text.
// With NullExtractor (default), episodes are stored but no facts are extracted.

const result = mem.store('Alice prefers dark mode and works at Acme Corp.');

// result: StoreResult
// {
//   episodeId:       string       — UUID of the stored episode
//   entityIds:       string[]     — UUIDs of entities created/updated
//   factIds:         string[]     — UUIDs of facts inserted
//   quarantinedCount: number      — facts below confidence threshold
// }

// Structured input (JSON object)
mem.store({ subject: 'Alice', predicate: 'role', object: 'Engineer' });

// With metadata and explicit scope
mem.store('Bob joined the team today.', {
  source: 'slack',
  timestamp: new Date().toISOString(),
}, 'workspace-7');

# Python
result = mem.store("Alice prefers dark mode.")
result = mem.store({ "subject": "Alice", "predicate": "role", "object": "Engineer" })
// retrieve() — ranked facts, entities, and source episodes

const result = mem.retrieve('Alice preferences');

// result: RetrievalResult
// {
//   facts:    Fact[]        — top-K ranked facts
//   entities: Entity[]      — entities mentioned in the top facts
//   episodes: Episode[]     — source episodes for the top facts
//   scores:   Record<id, number>  — final hybrid score per fact
//   traceIds: string[]      — IDs of the retrieval trace records written
// }

// With all options
const result = mem.retrieve(
  'Alice preferences',
  5,                                   // topK
  'workspace-7',                       // scope
  new Date('2025-01-01'),             // asOf — point-in-time query
);

// Inspect scores
for (const fact of result.facts) {
  console.log(fact.predicate, '→', result.scores[fact.id].toFixed(3));
}

# Python
result = mem.retrieve("Alice preferences", top_k=5)
for fact in result.facts:
    print(fact.predicate, fact.object_value_json)
// getContext() — LLM-ready formatted context string + structured data
// Includes memory blocks (always-pinned) + ranked facts + entities

const ctx = mem.getContext('What does Alice prefer?');

// ctx: ContextResult
// {
//   formatted: string    — ready to inject into an LLM prompt
//   facts:     FactView[]
//   entities:  EntityView[]
//   blocks:    Record<string, string>   — always-pinned memory blocks
//   timeline:  TimelineEvent[]          — ordered history
// }

// Inject directly into a system prompt
const systemPrompt = `
You are a helpful assistant.

## Long-term memory
${ctx.formatted}

Answer based on the above context.
`;

# Python
ctx = mem.get_context("What does Alice prefer?")
system_prompt = f"...\n\n## Memory\n{ctx.formatted}"
// updateFact() — supersede a fact with a new value
// The old fact gets truth_state='superseded', valid_to=now.
// The new fact gets truth_state='current', valid_from=now.

const { facts } = mem.retrieve('Alice theme');
const oldFact = facts.find(f => f.predicate === 'prefers');

const newFact = mem.updateFact(oldFact.id, 'light mode');

// Point-in-time query — see what we knew BEFORE the update
const historical = mem.retrieve('Alice theme', 5, undefined, yesterday);
// historical.facts[0].objectValueJson.value === 'dark mode'

# Python
new_fact = mem.update_fact(old_fact.id, "light mode")
// forget() — soft hide or hard delete
// Soft (default): truth_state='hidden', still queryable via asOf
// Hard: permanently removed from the database

// Forget a single fact (soft)
mem.forget({ factId: 'uuid-of-fact' });

// Forget a single fact (hard delete)
mem.forget({ factId: 'uuid-of-fact' }, true);

// Forget ALL facts about an entity
mem.forget({ entityId: 'uuid-of-entity' });

// Forget everything from a specific episode (e.g. bad data import)
mem.forget({ episodeId: 'uuid-of-episode' });

// Wipe an entire scope
mem.forget({ scopeId: 'temp-session-99' }, true);

// forget() returns ForgetResult:
// { affectedFacts: number, affectedEntities: number }

# Python
mem.forget(fact_id="uuid")
mem.forget(entity_id="uuid", hard=True)
// Memory blocks — always-surfaced pinned key/value state
// Unlike retrieved facts, blocks appear in EVERY getContext() call.
// Use for: persona, current goal, user profile, agent config.

mem.setBlock('persona', 'You are a concise technical assistant.');
mem.setBlock('user_name', 'Alice');
mem.setBlock('current_goal', 'Finish the Q2 roadmap.');

const goal = mem.getBlock('current_goal');
// 'Finish the Q2 roadmap.'

const all = mem.getBlocks();
// { persona: '...', user_name: 'Alice', current_goal: '...' }

mem.deleteBlock('current_goal');

// Blocks are scoped too
mem.setBlock('plan', 'Deploy to staging', 'session-abc');

# Python
mem.set_block("persona", "You are a helpful assistant.")
mem.get_block("persona")
mem.get_blocks()
// Background workers — call periodically (cron, timer, etc.)

const workers = mem.workers;

// 1. Decay — reduce strength of infrequently accessed facts
workers.runDecay('default');

// 2. Heat promotion — find hot facts (access × strength / log(age))
const hotIds = workers.runHeatPromotion('default');
// returns string[] of fact IDs above heatPromotionThreshold

// 3. Reconsolidation — re-evaluate quarantined facts
// If a fact was quarantined (low confidence) but the same
// subject+predicate was later confirmed, reject the quarantine.
workers.runReconsolidation('default');

// 4. Reinforce accessed facts
workers.reinforce(['fact-uuid-1', 'fact-uuid-2']);

// 5. Full cleanup cycle
workers.runCleanup('default');

// Cron example (runs every 24 hours)
setInterval(() => {
  workers.runDecay('default');
  workers.runReconsolidation('default');
}, 24 * 60 * 60 * 1000);

# Python
mem.workers.run_decay("default")
mem.workers.run_reconsolidation("default")
// Custom Extractor — bridge between text and structured facts
// Engram ships with NullExtractor (no-op). Wire in your LLM or NLP pipeline.

import { BaseExtractor } from 'engram-node';

class LLMExtractor extends BaseExtractor {
  extract(data) {
    // Call your LLM or NLP model here
    return {
      entities: [
        { name: 'Alice',    type: 'person', confidence: 0.98 },
        { name: 'Acme Corp', type: 'org',    confidence: 0.95 },
      ],
      facts: [
        {
          subject:    'Alice',
          predicate:  'works_at',
          object:     'Acme Corp',
          confidence: 0.95,
          factType:   'assertion',
        },
      ],
      confidence: 0.95,
    };
  }
}

mem.setExtractor(new LLMExtractor());
mem.store('Alice works at Acme Corp.');

# Python — same pattern
from engram import BaseExtractor

class MyExtractor(BaseExtractor):
    def extract(self, data):
        return {"entities": [...], "facts": [...], "confidence": 0.9}

mem.set_extractor(MyExtractor())
// Scopes — namespace isolation for multi-user / multi-agent systems
// Every API method accepts an optional scope parameter.
// Different scopes share the same SQLite file but never cross-contaminate.

const memAlice = new Engram({ dbPath: 'shared.db', defaultScope: 'alice' });
const memBob   = new Engram({ dbPath: 'shared.db', defaultScope: 'bob'   });

memAlice.store('Alice likes coffee.');
memBob.store('Bob prefers tea.');

// Cross-scope queries return nothing
const r = memBob.retrieve('coffee');
// r.facts.length === 0  ✓ Alice's data is invisible to Bob's scope

// Or pass scope explicitly per call
mem.store('Alice likes coffee.', {}, 'alice');
mem.retrieve('coffee', 5, 'alice');
mem.setBlock('goal', 'Ship v1', 'alice');

# Python
mem_alice = Engram(db_path="shared.db", default_scope="alice")
mem_bob   = Engram(db_path="shared.db", default_scope="bob")
Real-world use cases

Where you'd actually use Engram

💬

Persistent chatbot memory

Remember user preferences across sessions

// User says: "I prefer short answers"
mem.store('User prefers concise replies.');

// Next session — inject memory into prompt
const ctx = mem.getContext('user style preferences');
// ctx.formatted → "User prefers concise replies."
systemPrompt += ctx.formatted;
🤖

Autonomous agents

Agents that learn and update their world model

// Agent observes the world
agent.store('Server CPU is at 94%.');
agent.store('Deployment pipeline is blocked.');

// Agent recalls relevant context before acting
const ctx = agent.getContext('infrastructure status');
// Graph traversal surfaces both CPU + pipeline facts
🏥

Medical / legal record agents

Track entities and supersede outdated facts

// Patient record update
const r = mem.retrieve('blood pressure medication');
const old = r.facts.find(f => f.predicate === 'medication');

// Doctor updates prescription
mem.updateFact(old.id, 'lisinopril 10mg');

// Historical query still returns old fact
mem.retrieve('medication', 5, undefined, asOf);
📊

CRM / sales intelligence

Entity graph links contacts, companies, deals

// Store relationship facts
mem.store('Alice works at Acme. Bob manages Alice.');
mem.store('Acme has a $500K open deal.');

// Querying "Bob" surfaces Acme + deal via graph
const r = mem.retrieve('Bob', 10);
// 2-hop BFS: Bob → Alice → Acme → deal
🎓

Tutoring / learning systems

Track what the learner knows and forgets

// Student mastery tracking
mem.setBlock('weak_topics', 'recursion, pointers');

// After quiz — update mastered topics
mem.store('Student mastered recursion.');

// Next session — personalised question selection
const ctx = mem.getContext('student knowledge');
// Blocks + facts surface weak areas automatically
🔧

DevOps / incident response

Incident timeline with point-in-time queries

// Alert fires
mem.store('service-auth: latency_p99 is 4200ms', {}, 'incident-42');
mem.store('service-auth: depends_on postgres-primary', {}, 'incident-42');

// Post-mortem — query what we knew at T+5m
const snapshot = mem.retrieve('auth latency', 20, 'incident-42', t5m);
// workers.runDecay() cleans stale incident facts over time
